📄 Project Description: DevAll
DevAll is a production-grade, decoupled, microservices-based developer portfolio aggregator designed to unify a developer's fragmented digital identity. Instead of recruiters jumping between GitHub, LeetCode, Codeforces, and Hugging Face, DevAll provides a verified, real-time dashboard that consolidates technical stats, project repositories, and competitive programming rankings into a single, professional link.
Core Value Proposition:
* Verification: Data is pulled directly via official APIs, eliminating "resume padding."
* Consolidation: A "One-Stop-Shop" for a developer’s entire ecosystem (Logic, Code, AI/ML, and Social).
________________


⚙️ Technical Workflow (The Lifecycle of a Sync)
Step 1: Entry & Authentication
* Trigger: The user logs in via Next.js using GitHub OAuth.
* Gateway: All requests hit the Kong API Gateway. It verifies the JWT and checks Redis to ensure the user isn't spamming the "Sync" button (Rate Limiting).
* Auth Service: The NestJS Auth Service confirms the user’s identity and permissions.
Step 2: Asynchronous Job Dispatch
* Event: When the user clicks "Refresh My Stats," the Profile Service (NestJS) does not call the external APIs directly. Instead, it pushes a SYNC_TASK message into RabbitMQ.
* UI State: The frontend receives a 202 Accepted status immediately, and TanStack Query puts the dashboard into a "Syncing..." state.
Step 3: Distributed Data Collection
* Processing: The Collector Service (Python/FastAPI) acts as a worker. It pulls the task from the queue.
* Ingestion: It concurrently hits multiple endpoints (GitHub GraphQL, LeetCode REST, etc.).
* Normalization: It uses Pydantic to clean the messy, varied data from these sources into a standardized DevAll Schema.
Step 4: Data Persistence & Synchronization
* Internal Comm: Once data is cleaned, the Python service sends the payload back to the Profile Service using gRPC for maximum speed.
* Storage: The Profile Service updates PostgreSQL (the source of truth) and invalidates the old Redis cache.
Step 5: Real-time Update
* Finalization: The Profile Service emits a "Sync Complete" event.
* Display: The Next.js frontend (via WebSockets or polling) detects the completion and refreshes the UI, showing the updated "Unified Heatmap" and new achievements.

🛠️ Tech Stack Cheat Sheet
* Frontend: Next.js 15, TypeScript, TanStack Query, Tailwind.
* Backend: NestJS (Node.js) & FastAPI (Python).
* Gateway: Kong / Nginx.
* Messaging: RabbitMQ (AMQP).
* Inter-Service: gRPC + Protocol Buffers.
* Database: PostgreSQL (Primary), Redis (Cache/Rate Limit).
* DevOps: Docker, GitHub Actions (CI/CD).

🚀 DevAll: Official Tech Stack Summary

1. Frontend (The User Interface)
* Framework: Next.js 15+ (App Router).
* Language: TypeScript.
* State Management: TanStack Query (React Query) for server-state & fetching.
* Styling: Tailwind CSS + Shadcn UI (Industry standard for dashboards).
* Visualization: Recharts or Framer Motion (for those sleek profile heatmaps).

2. Backend Services (Microservices)
We are using a Polyglot (multi-language) approach to use the best tool for each job.
Layer
	Technology
	Entry Point-Kong API Gateway
	Frontend-Next.js
	Auth Service - Node.js / NestJS
	Profile Service - Node.js / NestJS
	Collector Service - Python / FastAPI
	Messaging - RabbitMQ
	Internal Comm - gRPC
	ORM - Prisma, Pydantic
	Persistence - PostgreSQL & Redis
	Metrics - Prometheus, Grafana
	

3. Infrastructure & Communication (The "Glue")
This is where the "Senior Engineer" vibes come from.
* Inter-Service Comm: gRPC (Protocol Buffers). Fast, binary communication between Node and Python.
* Message Broker: RabbitMQ. Handles asynchronous tasks (e.g., "User clicked sync, start background scraping").
* Database: PostgreSQL. The relational "Source of Truth."
* Caching: Redis. Used for rate-limiting (external APIs) and caching public profiles for sub-100ms loads.

4. DevOps & Tooling
* Containerization: Docker & Docker Compose (essential for local dev).
* ORM: Prisma (for Node.js) and SQLAlchemy (for Python).
* Validation: Zod (Node) and Pydantic (Python) to ensure data integrity.
* API Docs: Swagger/OpenAPI (automatically generated by FastAPI and NestJS).

🏗️ The "DevAll" Architecture in a Nutshell
1. Next.js asks the Profile Manager (Node) for a profile.
2. Profile Manager checks Redis. If not there, it checks Postgres.
3. If the data is old, the Profile Manager sends a "Sync" event to RabbitMQ.
4. The Data Collector (Python) sees the event, scrapes the latest stats, and sends them back via gRPC.
5. Postgres is updated, Redis is cleared, and the user sees the fresh data instantly.
